Remarks
There are several limitations in the current implementation of this operation.
  • If more than one assignment of values to variables exists which is capable of producing the candidate URI through the BindByName Overload operation, it is unspecified which assignment of values is chosen.
  • Simple string values will always be returned as a String. Associative array values will always be returned as a IList T  whose values are of type String. Associative map values will always be returned as a IDictionary TKey, TValue  whose keys and values are of type String. No other deserialization or coercion of values is performed by this library.

The matching algorithm prefers to use simple string values for all variables not explicitly listed in arrayVariables or mapVariables. If no assignment of values to variables is possible using this choice, one or more variables may be treated as associative arrays and/or maps in order to produce a successful assignment. The exception to this rule is compound template variables (which use the explode modifier); these variables prefer to match as arrays instead of simple strings, even if the result produces an array containing exactly one string.
